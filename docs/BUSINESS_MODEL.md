# 1. 비즈니스 모델 철학

## 1.1 LV UP에 돈을 내는 이유

LV UP의 비즈니스 모델은 **"기능을 파는 것"**이 아니라 **"자기 자신에게 한 약속의 증표를 주는 것"**에 가깝다.

대부분의 SaaS는 기능을 판다. 더 많은 저장 공간, 더 빠른 처리, 더 많은 사용자. 사람들은 도구가 더 편리해지면 돈을 낸다.

LV UP은 다르다. LV UP은 **자기 자신이 플레이어가 되기로 한 약속**의 시스템이다. 사람들이 돈을 내는 이유는 (우선순위 순):

1. **자신의 변화에 진지함을 표시하기 위해** — 결제는 자기 자신과의 계약
2. **자기주도의 도구를 해금하기 위해** — 무제한 커스텀 퀘스트, 캐릭터 코스, 자기만의 시스템 설계 권한
3. **같은 길을 가는 사람들을 곁에 두기 위해** — 사회적 동행
4. **창업자와 직접 소통하기 위해** — 희소한 접근권

### 왜 이 순서인가

이 순서가 뒤집히면 LV UP은 "함께 모이는 컬트"로 흐른다. 
1, 2번이 내부 지향(자기와의 계약, 자기주도 도구)이고 3, 4번이 외부 지향(동행, 접근권).
**내부가 외부보다 먼저 와야 운영원칙 0번(떠날 수 있는 시스템)이 지켜진다.**

특히 2번이 핵심이다. 무료 사용자는 시스템이 제공하는 퀘스트를 받기만 하고(수동), 
유료 사용자는 자기 퀘스트를 무제한 만들 수 있다(자기주도). 
결제가 곧 "관찰자에서 플레이어로"의 시스템적 전환이다.

기능 자체는 부차적이다. 심지어 **핵심 기능은 무료가 더 유리**할 수 있다. 
사람이 많아야 LV UP의 메시지가 퍼지니까.

## 1.2 핵심 원칙

**원칙 1: 무료가 기본, 유료는 더 깊은 관계**
- 핵심 기능은 영원히 무료
- 유료는 도구가 아니라 관계와 정체성을 산다

**원칙 2: 첫 100명은 기능이 아니라 사람이 모은다**
- 광고로 모은 사람은 광고로 떠난다
- 입소문, 직접 초대, 진심의 소통이 첫 100명을 모은다

**원칙 3: 가격은 가치보다 약속이 만든다**
- 가격이 너무 낮으면 진지하지 않게 받아들임
- 가격이 너무 높으면 진입장벽이 됨
- 적절한 가격은 "이걸 산 나는 진지하다"를 표시할 수 있는 수준

**원칙 4: 검증되지 않은 모델은 팔지 않는다**
- 작동하는지 모르는 걸 팔면 첫 사용자에게 사기다
- 무료로도 사람이 안 모이면 유료로는 절대 안 모인다
- 매출은 검증의 결과이지 검증의 수단이 아니다

**원칙 5: 다층 수익은 한 번에 만들지 않는다**
- 한 시점에 하나의 수익원만 검증
- 검증된 후에 다음 레이어 추가
- 5개 레이어를 동시에 만들면 모두 망한다

## 1.3 비즈니스 모델의 진화 단계

LV UP의 비즈니스 모델은 다음 순서로 진화한다.

```
Phase 0 (M1~M3)   : 무료, 사용자 검증
       ↓
Phase 1 (M3~M6)   : Founder's Circle (월 멤버십)
       ↓
Phase 2 (M6~M12)  : 시즌 코호트 (4주 챌린지)
       ↓
Phase 3 (M12~M18) : 다층 수익 (멤버십 + 코호트 + B2B)
       ↓
Phase 4+ (Y2~)    : 큰 그림 (IP, 아카데미, 공간, 투자)
```

각 Phase는 이전 Phase에서 검증된 수치 위에서 시작한다.
검증되지 않으면 다음 Phase로 넘어가지 않는다.

# 2. Phase 0: 검증 단계 (M1~M3)

## 2.1 목표

**매출 0원, 대신 핵심 지표 검증**

이 단계에서 매출을 만들려고 시도하면 안 된다.
이유: **작동하는지도 모르는 것을 파는 것은 첫 사용자에게 사기다.**

## 2.2 검증할 가설

| 가설 | 검증 지표 | 목표 |
|---|---|---|
| 사람들이 LV UP을 이해하고 흥미를 느낀다 | 가입 전환율 (랜딩 → 가입) | 5% 이상 |
| 가입한 사람이 첫 행동을 한다 | 가입 → 첫 퀘스트 완료율 | 60% 이상 |
| 첫 행동한 사람이 다음 날 돌아온다 | D1 재방문율 | 40% 이상 |
| 일주일 후에도 살아있는 사용자가 있다 | D7 재방문율 | 25% 이상 |
| 사용자가 LV UP을 친구에게 말한다 | 자발적 초대 비율 | 측정 가능한 수준 |

이 5가지가 충족되지 않으면 Phase 1로 넘어가지 않는다.
충족되지 않는다는 건 **"지금 만든 것이 사람들이 원하는 것이 아니다"**라는 뜻이고, 그 상태에서 유료화해도 망한다.

## 2.3 사용자 확보 전략

**광고 ❌, 마케팅 ❌, 입소문만**

베타 사용자를 모으는 채널:

1. **창업자의 직접 초대** (1순위)
   - 친한 친구 5~10명에게 직접 메시지
   - "내가 만들고 있는데 같이 해볼래?"
   - 이게 가장 솔직한 첫 사용자

2. **친구의 친구** (2순위)
   - 첫 사용자에게 "1명 초대해줘" 요청
   - 누구를 초대할지 직접 고르게 함 (아무나 X)

3. **작은 커뮤니티** (3순위)
   - 디스코드, 카카오톡 오픈채팅, X(트위터) 등
   - 자기계발/창업/개발 관련 작은 그룹
   - "이거 만들어봤어요" 톤으로 공유 (광고 X)

**목표 인원**: M3 시점에 베타 사용자 30~100명

## 2.4 비용 구조 (Phase 0)

**총 운영비: 월 5만원 이하**

| 항목 | 비용 | 비고 |
|---|---|---|
| 도메인 (lvup.world) | 월 약 3,000원 | 연 32달러 / 12 |
| Cloudflare (Pages, D1, R2, Email) | 0원 | 무료 tier 충분 |
| GitHub | 0원 | Public 무료 |
| Claude Code | 월 2~5만원 | API 사용량 또는 Pro 구독 |
| 기타 도구 | 0원 | 필수 외 안 씀 |

**핵심: Phase 0에서 돈을 많이 쓰면 안 된다.** 검증 안 된 것에 자본을 투입하는 건 가장 큰 실수다.

## 2.5 Phase 0 종료 기준

다음 모두 충족 시 Phase 1으로 진입:

- [ ] 베타 사용자 30명+ 모집
- [ ] 가입 → 첫 퀘스트 완료율 60%+
- [ ] D7 재방문율 25%+
- [ ] 사용자 중 최소 5명이 "이거 친구한테 말했어요" 자발적 보고
- [ ] 창업자 본인이 매일 LV UP을 사용하고 있음

**중요**: 5번이 가장 중요하다. 만든 사람이 안 쓰는 것을 다른 사람이 쓸 리 없다.

# 3. Phase 1: Founder's Circle (M3~M6)

## 3.1 목표

**첫 유료 사용자 10~30명, MRR 10~100만원**

이 Phase의 목표는 큰 매출이 아니다. **"사람이 LV UP에 돈을 내는가?"**라는 가장 본질적인 질문에 답하는 것이다.

## 3.2 상품 정의: Founder's Circle

**한 가지만 판다.** 여러 패키지 만들지 않는다.

| 항목 | 내용 |
|---|---|
| 이름 | Founder's Circle (또는 "창세 멤버십") |
| 가격 | 월 9,900원 또는 연 99,000원 (얼리버드 17% 할인) |
| 정원 | 첫 시즌 30명 한정 |
| 대상 | LV UP의 시작에 함께하고 싶은 사람 |

### 멤버십 혜택

혜택 순서가 중요하다. 결정 6(가치 제안 순서 뒤집기)에 따라 
**자기주도 도구 → 덕훌 코스 → 하프 → 동행 → 창업자 접근** 순.
내부 지향(자기와의 계약, 자기주도)이 외부 지향(동행, 접근권)보다 먼저 온다.

**1. 자기주도 도구의 해금** (가장 중요)
- 무료: 시스템이 제공하는 기본 퀘스트 + 커스텀 퀘스트 5개 제한
- **Founder: 커스텀 퀘스트 무제한 + 모든 도전 레벨(한계 시험까지) 접근**
- 이게 "관찰자에서 플레이어로"의 시스템적 전환점이다
- 결제 = 자기 시스템 설계 권한 획득

**2. 덕훌 성장 코스** (결정 4, 11)
- 소설 주인공 덕훌이 LV.0 → LV.5까지 통과한 퀘스트를 1:1로 따라 걷는 코스
- 퀘스트 완료 시 소설 외전 단락이 보상으로 해금
- **"소설을 더 깊이 읽는 유일한 방법"**
- 향후 허민 코스, 마지석 코스 등 다른 캐릭터 코스 확장 예정
- Founder's Circle 전용

**3. 하프와의 첫 만남** (결정 10, Phase 1 후반 도입)
- 무료 사용자: 익명 시스템 메시지 (작가가 썼지만 발신자 미공개)
- **Founder: 하프(HARP)라는 이름 명시, 하프가 직접 말 걸어줌**
- Phase 1 후반에 100개 메시지 풀로 시작 → Phase 2에 300개 → Phase 3에 500개
- Phase 3부터는 유료 LLM 대화 기능 (Founder 전용, 일일 제한)

**4. 전체 기능 사용**
- 무료는 일부 제한 (예: 캐릭터 코스, 외부 데이터 노출)
- 멤버는 무제한
- 단, **핵심 루프(가입→퀘스트→인증→보상)는 영원히 무료**

**5. 같은 길을 가는 사람들 — 비공개 디스코드**
- 멤버 전용 채널
- 같은 시즌을 걷는 동료들
- 랭킹/비교 없음, **"보이는 동행, 보이지 않는 부재"** 원칙 (3.7 참조)
- 신규 기능 미리 공유

**6. 월 1회 라이브 회고**
- 창업자가 진행
- 그달의 LV UP 진행 상황 공유
- 멤버들의 질문과 피드백
- 줌 또는 디스코드 음성

**7. 후원자 배지 + 상태창 특별 표시**
- 닉네임 옆에 "Founder" 배지
- 상태창에 가입 일자 표시
- "n번째 Founder"라는 희소성

**8. 신규 기능 베타 우선 접근**
- 모든 신기능을 먼저 사용
- 의견이 실제로 반영됨

**9. 평생 혜택 약속**
- 가격은 오를 수 있지만, **첫 30명은 평생 같은 가격**
- 이게 강력한 동기

## 3.3 왜 이 구조가 작동하는가

**1. 도구가 아니라 정체성을 판다**
사람들이 9,900원에 사는 건 기능이 아니다. **"LV UP의 첫 30명 중 한 명"**이라는 정체성이다.

**2. 가격이 적절하다**
- 너무 낮으면 진지하지 않게 받아들임
- 너무 높으면 진입 장벽
- 9,900원은 "이거 한 잔 커피값이네"라고 부담 없이 결제할 수 있는 수준
- 동시에 "내가 결제했으니 진짜 써야지"라는 약속 효과

**3. 창업자 직접 소통이 핵심 가치**
이건 100명 넘으면 못 한다. **그래서 희소함**. 30명만 가능한 경험.

**4. 평생 가격 약속이 결정적**
"지금 안 들어가면 나중엔 비싸진다"는 손실 회피 심리.
이건 거짓말이 아니라 **실제로 가격을 올릴 것**이라는 약속.

## 3.4 출시 전략

4. 출시일 동시 오픈

1. lvup.world에 "Founder's Circle 모집" 공지
2. genesis@lvup.world로 결제 안내
3. 디스코드 채널 오픈
4. 첫 24시간 한정으로 추가 혜택 (예: "1번 Founder" 칭호는 가장 먼저 결제한 사람만)

# 4. Phase 2: 시즌 코호트 (M6~M12)

## 4.1 목표

**첫 진짜 매출, MRR 100~500만원, 누적 사용자 500~1,000명**

Phase 1이 검증되면, 매출을 본격적으로 만들 수 있는 시점이다.

## 4.2 상품 정의: 시즌 코호트

시즌 코호트는 두 단계로 나뉜다. 결정 7(시즌 4주+4주 분리)에 따라:
- **시즌 1 (기초 4스탯)**: 자기와의 게임 — 체력/집중력/실행력/지식력
- **시즌 2 (후 2스탯 + 통합)**: 세상과의 게임 — 관계력/전파력

이 분리는 매슬로우 사다리(자기→타인)와 일치하고, 
LV UP 철학("자기 다룰 줄 모르면 타인을 다룰 수 없다")과도 맞는다.
시즌 1 졸업이 시즌 2의 강제 조건은 아니지만 권장 순서다.

### 시즌 1 — "첫 30일 챌린지: 자기와의 게임"

| 항목 | 내용 |
|---|---|
| 이름 | LV UP 시즌 1: 자기와의 게임 (첫 30일) |
| 가격 | 1인당 99,000원 (4주) |
| 정원 | 첫 시즌 15명 → 다음 30명 → 50명 |
| 기간 | 4주 (월요일 시작 → 일요일 종료) |
| 운영 빈도 | 분기 1회 (Q1, Q2, Q3, Q4) |
| 핵심 스탯 | 체력, 집중력, 실행력, 지식력 |

**시즌 1 구성:**

- **Week 1: 체력의 주** — 몸이 모든 능력의 그릇이다
  - 일일 퀘스트: 운동, 산책, 수면, 식습관
  - 주말 미션: 한 주의 변화 기록
- **Week 2: 집중력의 주** — 산만한 세상에서 한 점에 머물기
  - 일일 퀘스트: 딥워크, 디지털 디톡스, 몰입 시간
  - 주말 미션: 가장 집중했던 순간 기록
- **Week 3: 실행력의 주** — 생각만 하는 자와 실행하는 자 사이의 우주
  - 일일 퀘스트: 미루던 일 시작, 작은 결과물 만들기
  - 주말 미션: 한 가지 완성
- **Week 4: 지식력의 주** — 배우고 이해하고 해석하는 힘
  - 일일 퀘스트: 독서, 강의 수강, 학습 공유
  - 주말 미션: 시즌 졸업 회고 + 자기 학습 지도 작성

### 시즌 2 — "두 번째 30일: 세상과의 게임"

| 항목 | 내용 |
|---|---|
| 이름 | LV UP 시즌 2: 세상과의 게임 |
| 가격 | 1인당 99,000원 (4주) |
| 정원 | 시즌 1과 동일 |
| 기간 | 4주 |
| 운영 빈도 | 분기 1회 |
| 핵심 스탯 | 관계력, 전파력 + 6스탯 통합 |
| 권장 선행 | 시즌 1 졸업 (필수 아님) |

**시즌 2 구성:**

- **Week 1: 관계력의 주** — 사람과 건강하게 연결되는 힘
  - 일일 퀘스트: 협업, 피드백 주고받기, 조율
  - 주말 미션: 한 사람의 변화에 기여한 기록
- **Week 2: 전파력의 주** — 생각과 결과물을 다른 사람에게 전달하는 힘
  - 일일 퀘스트: 글쓰기, 발표, 콘텐츠 공유
  - 주말 미션: 자기 노하우를 한 사람에게 전달
- **Week 3: 관계력+전파력 결합의 주** — 영향력의 윤리
  - 일일 퀘스트: 커뮤니티 기여, 피드백 루프
  - 주말 미션: 내 영향력의 방향 점검
- **Week 4: 6스탯 통합의 주** — 균형과 편향
  - 일일 퀘스트: 자기 강점 극대화, 약점 수용
  - 주말 미션: 시즌 2 졸업 회고 + 자기 정체성 선언

### 시즌 운영

**1. 같은 시즌, 같은 시작 — "보이는 동행, 보이지 않는 부재"**

결정 8의 디자인 원칙:

- 모든 참여자가 같은 월요일에 시작
- 전체 디스코드 단톡방 운영
- **매일 누가 무엇을 했는지는 보여주되, 누가 무엇을 안 했는지는 보여주지 않음**
  - 보이는 비교 (나쁨): "오늘 25명 중 18명이 인증했습니다. 당신은 안 했습니다."
  - 보이는 동행 (좋음): "오늘 18명이 같은 길을 걸었습니다."
- 후자는 동행감이고, 전자는 압박이다. 전자는 운영원칙 0번을 위반한다.
- **사회적 동행 = 가장 강한 동기. 사회적 압력 ≠ LV UP의 방식.**

**2. 매주 라이브 회고**
- 매주 일요일 저녁 8시 (KST)
- 창업자 진행
- 한 주의 변화 공유
- 다음 주 미션 선언

**3. 시즌 종료 졸업식**
- 4주 후 졸업식 (라이브)
- 시즌 졸업 뱃지 부여
- 가장 인상적인 변화 5명 선정 → 인터뷰 콘텐츠 (본인 동의 시)
- 다음 시즌 신청 안내
- **"졸업"이 곧 "떠나도 괜찮음"의 증명** — 운영원칙 0번과 연결

## 4.3 왜 이 모델이 강한가

**1. 사회적 동행**
혼자 시작하면 3일 만에 그만둔다. 같이 시작하면 4주를 채운다.
이건 검증된 인간 행동 패턴이다 (부트캠프, 헬스장, 학원이 다 이걸로 작동).

**2. 시간 제한이 결제 동기를 만든다**
"언제든 시작 가능"은 결제하지 않는다. "이번 주 일요일까지 신청 마감"은 결제한다.

**3. 명확한 결과 약속**
4주 후 무엇이 달라질지 약속할 수 있다.
→ "4주 후 당신은 매일 한 가지 작은 행동을 인증하는 사람이 되어 있을 것이다"

**4. 반복 가능**
시즌 1, 2, 3, 4... 무한 확장 가능.
시즌마다 주제 변형 가능 (예: 시즌 2 = 창작자의 시즌, 시즌 3 = 프로젝트 출시 시즌)

**5. 검증된 모델**
부트캠프, 챌린지 프로그램, 코호트 기반 코스가 다 이 모델로 작동.
LV UP의 차별점은 **"끝나도 LV UP 앱은 계속 사용한다"**는 것 — 시즌이 평생 사용자를 만드는 입구가 됨

## 4.4 시즌 코호트의 단위 경제

| 항목 | 가정 |
|---|---|
| 시즌당 정원 | 30명 (안정화 후) |
| 시즌당 가격 | 99,000원 |
| 시즌당 매출 | 297만원 |
| 시즌 운영 비용 | 약 30만원 (디스코드 부스트, 디자인, 졸업 굿즈) |
| 시즌당 순매출 | 약 267만원 |
| 분기 1회 운영 | 연 4회 |
| 연 시즌 매출 | 약 1,068만원 |

여기에 Founder's Circle MRR 100만원 × 12 = 1,200만원 합치면 **연 약 2,268만원**.

이건 **1인 운영 가능한 규모**이고, LV UP의 다음 단계 (아카데미, 공간) 자금의 시드가 된다.

## 4.5 시즌 운영의 한계

**창업자가 매번 진행하면 확장 불가**

- 시즌 1~4까지는 창업자가 직접 진행 (M6~M12)
- 시즌 5부터는 시즌 졸업생 중 우수자를 멘토로 양성 (Phase 3)
- 시즌 10부터는 멘토 시스템이 핵심 운영 (M18~)

이게 자연스럽게 다음 단계 "아카데미"로 연결되는 길이다.

## 4.6 Phase 2 종료 기준

- [ ] 시즌 코호트 4회 이상 운영 (분기마다)
- [ ] 누적 시즌 졸업생 100명 이상
- [ ] 시즌 졸업률 70%+ (4주 끝까지 가는 비율)
- [ ] 졸업생 중 30%+ 가 다음 시즌 또는 Founder's Circle 전환
- [ ] 누적 매출 1,000만원 이상
- [ ] 멘토로 양성 가능한 졸업생 3명 이상 발견

# 5. Phase 3: 다층 수익 구조 (M12~M18)

## 5.1 목표

**MRR 500~2,000만원, 안정적 수익원 3개 이상**

Phase 2까지 검증된 두 가지 수익원 위에 새로운 레이어를 추가한다.

## 5.2 동시 운영 3가지

### 1. 무료 LV UP 앱
- 핵심 기능 영원히 무료
- 광고 없음
- 데이터 판매 없음
- 사용자 확보 + 운동 확산의 핵심

### 2. Founder's Circle (Phase 1에서 진화)
- 가격 인상: 월 9,900원 → 19,900원 (신규 가입자만)
- 첫 30명은 평생 9,900원 유지 (약속 지키기)
- 멤버 수: 100~300명 목표
- MRR 200~600만원

### 3. 시즌 코호트 (Phase 2에서 진화)
- 가격 인상: 99,000원 → 149,000원
- 정원 확대: 30 → 50 → 80명
- 멘토 시스템 도입 (졸업생이 진행)
- 분기 1회 → 분기 2회 → 월 1회로 빈도 증가
- 연 매출 3,000만원+

## 5.3 새로 추가할 실험

### 실험 A: B2B 워크숍

| 항목 | 내용 |
|---|---|
| 이름 | LV UP for Teams |
| 가격 | 1회 200~500만원 (기업 규모별) |
| 대상 | 스타트업, 중소기업 팀빌딩 |
| 내용 | 4시간~1일 워크숍, LV UP 시스템으로 팀 동기부여 + 1개월 팔로업 |
| 빈도 | 분기 1~2회 |

**왜 이게 가능한가**
- LV UP의 5클래스 시스템은 팀 진단 도구로 자연스럽게 작동
- 기업은 직원 동기부여에 큰 돈을 쓸 의향이 있음
- 1회 200만원 = 시즌 코호트 20명분 매출

**검증 방법**
- Phase 2 후반에 1~2개 회사에 무료/저가로 시범 운영
- 반응 좋으면 정식 출시

### 실험 B: LV UP 굿즈

| 항목 | 내용 |
|---|---|
| 종류 | 한정판 노트, 스티커, 머그컵, 티셔츠 |
| 가격 | 1만원~5만원 |
| 목적 | **수익보다 브랜드 강화** |
| 운영 | 시즌마다 한 번씩 한정 출시 |

**왜 이게 가능한가**
- LV UP은 운동이고, 운동에는 굿즈가 따라온다
- 굿즈를 산다는 건 정체성 표현
- 부수입이지만 **브랜드 인지도 + 사용자 충성도 강화**

### 실험 C: 1:1 코칭 (매우 한정)

| 항목 | 내용 |
|---|---|
| 가격 | 회당 30~50만원 |
| 정원 | 월 5명 한정 |
| 진행자 | 창업자 본인 |
| 내용 | 1시간 1:1 줌, LV UP 기반 인생 코칭 |

**왜 이게 가능한가**
- 가장 비싼 시간을 가장 비싸게 판다
- 정원 제한으로 희소성 + 가격 정당화
- 부수입이지만 **창업자의 통찰력 가치를 증명**

**한계**: 창업자 시간이 한정. 월 5명 이상 못 받음.
이건 **Phase 4로 가는 다리** — 나중에 멘토 시스템으로 확장.

## 5.4 Phase 3의 핵심 도전

**창업자 1명으로 운영 가능한 한계가 보이기 시작한다**

- 시즌 코호트 진행
- Founder's Circle 라이브
- B2B 워크숍 운영
- 1:1 코칭
- 동시에 제품 개발

**이때 결정해야 할 것**:
- 첫 직원 또는 공동 운영자를 받을 것인가?
- 멘토 시스템으로 위임할 것인가?
- 자동화로 해결할 수 있는가?

이 결정은 Phase 3 중간 (M15)쯤에 내려야 한다.

## 5.5 Phase 3 종료 기준

- [ ] MRR 500만원 이상

## 5.6 하프 시스템 Phase별 로드맵 (전 Phase 관통)

IDEAS.md의 1순위 시스템인 **하프(HARP) AI 동반자**는 LV UP의 비즈니스 
모델에서 가장 결정적인 차별화 장치다. Phase별로 어떻게 도입/확장되는지 
여기서 한눈에 정리.

상세한 시스템 정의는 PRD v5 Part 11.5~11.8 참조.

### Phase 0 (M1~M3): 메시지 풀 없음

- 검증 단계, 하프 시스템 비활성
- 익명 시스템 메시지만 사용
- 작가는 하프의 톤 가이드라인(PRD 11.7) 작성에 집중

### Phase 1 MVP (M3~M6): 메시지 풀 100개, 발신자 익명

- 작가가 메시지 풀 100개 작성 (welcome, reward, special 등 6개 카테고리)
- harp_messages 테이블에 저장 (PRD Part 19)
- **발신자 이름은 아직 노출 안 함** — 사용자 눈엔 익명이지만 작가가 쓴 자산
- 단계적 공개의 첫 단계

### Phase 1 후반 (M5~M6): 하프 이름 등장 (Founder's Circle 전용)

- Founder's Circle 멤버에게만 **"하프(HARP)"라는 이름 공개**
- 메시지 말미에 "— 하프" 서명 표시
- 메시지 풀 200개로 확장
- **이것이 결제 동기의 핵심 중 하나** — "Founder만 하프를 만난다"
- 무료 사용자는 계속 익명 메시지

### Phase 2 (M6~M12): 하프 캐릭터 전면 공개

- **모든 사용자에게 하프의 정체 공개** — 일러스트, 캐릭터 소개
- 메시지 풀 300개
- 소설과의 명시적 연결: "덕훌이 소설에서 만난 그 하프"
- 양방향 유입 본격 가동 (앱→소설, 소설→앱)
- Story Quest와 결합 — 하프가 덕훌이 받은 퀘스트를 사용자에게 제안

### Phase 3 (M12~M18): 유료 LLM 대화 (Founder's Circle 전용)

- Founder's Circle 멤버에게 **하프와의 직접 대화 기능** 해금
- 일일 제한 (예: 3회)
- 하프의 톤 가이드라인을 시스템 프롬프트로 사용
- AI 비용 주의: 매출의 30% 이하 유지 (IDEAS.md 비용 시나리오 참조)
- 주간 회고 시 하프의 행동 요약 — 유료 기능의 핵심 가치

### Phase 4+ (Y2~): 하프와 IP의 결합

- 하프 어록집 굿즈 (작가가 쓴 500개 메시지가 한 권의 책)
- 하프 캐릭터 일러스트 상품화
- 소설/웹툰/애니에서 하프의 정체성 확장
- 하프가 LV UP IP의 얼굴 중 하나로 자리잡음

### 왜 이 로드맵이 결정적인가

보통 IP-앱 관계는 "IP를 앱에서 재활용" 정도인데, LV UP은 다르다.
**앱의 핵심 기능(하프)이 곧 소설의 핵심 캐릭터이고, 
소설을 읽은 독자가 앱에서 그 캐릭터를 실제로 만난다.**

이게 IDEAS.md 1.4 "양방향 유입"의 가장 강력한 구현이다. 
그리고 이 로드맵이 Phase별로 안 박혀 있으면, 개발 중에 "하프는 나중에"로 
밀려 영원히 구현 안 된다. 여기 박아두는 건 **비즈니스 모델 차원의 약속**이다.

# 7. 단위경제 가설

## 7.1 사용자당 평균 매출 (ARPU)

**Phase 0 (M3)**
- 무료 사용자: 0원
- ARPU: 0원

**Phase 1 (M6)**
- 무료 사용자 200명, Founder 20명
- 매출: 20명 × 9,900원 × 6개월 = 약 120만원
- ARPU: 약 6,000원 (전체 사용자 기준)

**Phase 2 (M12)**
- 무료 500명, Founder 50명, 시즌 졸업생 누적 100명
- 매출: Founder 540만원/년 + 시즌 990만원/년 = 1,530만원
- ARPU: 약 23,500원 (활성 사용자 650명 기준)

**Phase 3 (M18)**
- 무료 2,000명, Founder 200명, 시즌 졸업생 누적 300명, B2B 5건
- 매출: Founder 약 4,000만원/년 + 시즌 약 4,500만원/년 + B2B 2,000만원/년 = 1억원+
- ARPU: 약 40,000원 (활성 사용자 2,500명 기준)

이 숫자들은 **가설**이다. 절대 약속이 아니다.

## 7.2 사용자 획득 비용 (CAC)

**Phase 0**: 0원 (직접 초대만)
**Phase 1**: 0원 (입소문 + 베타 사용자 추천)
**Phase 2**: 0~5,000원 (시즌 졸업생 콘텐츠가 마케팅이 됨)
**Phase 3**: 5,000~20,000원 (콘텐츠 마케팅 + SEO + 일부 광고 실험)

**핵심 원칙**: CAC가 ARPU의 1/3을 넘으면 그 채널은 중단.

## 7.3 LTV (생애 가치)

**Founder's Circle**: 평균 12개월 유지 가정 → LTV 약 12만원
**시즌 코호트 졸업생**: 평균 1.5회 재참여 가정 → LTV 약 15만원
**B2B 고객**: 평균 1.2회 반복 가정 → LTV 약 360만원

**LTV/CAC 비율 목표**: 3 이상 (건강한 SaaS의 기본 기준)

## 7.4 손익분기점 (BEP)

| Phase | 월 비용 | BEP 매출 | BEP 사용자 |
|---|---|---|---|
| Phase 0 | 5만원 | 0원 (검증 단계) | - |
| Phase 1 | 20만원 | 20만원 | Founder 20명 |
| Phase 2 | 50만원 | 50만원 | Founder 50명 또는 시즌 1회/분기 |
| Phase 3 | 300~500만원 (인건비 포함) | 500만원 | 다층 수익 안정화 |

**핵심**: Phase 1에서 BEP를 빠르게 달성하는 것이 가장 중요. 그래야 자본 없이도 다음 단계로 갈 수 있다.

# 8. 검증 지표와 의사결정 기준

## 8.1 어떤 지표를 볼 것인가

**Phase 0 (M1~M3)**
- 가입 → 첫 퀘스트 완료율
- D1 / D7 재방문율
- 자발적 입소문 (정성)
- 창업자 본인의 사용 여부

**Phase 1 (M3~M6)**
- 무료 → Founder 전환율
- Founder 월 이탈률
- 디스코드 활성도 (DAU)
- 라이브 회고 참석률
- "내 인생이 바뀌었다" 후기 수

**Phase 2 (M6~M12)**
- 시즌 신청 → 결제 전환율
- 시즌 4주 완주율 (졸업률)
- 졸업생 → Founder 또는 다음 시즌 전환율
- 시즌 NPS (Net Promoter Score)
- MRR 성장률

**Phase 3 (M12~M18)**
- 수익원별 MRR
- 신규 사용자 채널별 CAC
- 코호트별 LTV
- 운영 자동화 비율
- 창업자 시간 사용 분포

## 8.2 의사결정 기준

### 다음 Phase로 갈 것인가?

각 Phase 종료 기준이 충족되면 → 진입
충족되지 않으면 → **Phase에 머물면서 가설 수정**, 다음 Phase로 가지 않음

### 어떤 기능을 만들 것인가?

기준:
1. 핵심 루프(행동→인증→보상)에 필수인가?
2. Founder's Circle 멤버가 명시적으로 요청했는가?
3. 시즌 코호트 진행에 필요한가?
4. 위 셋이 아니면 → **만들지 않음**

### 어떤 기능을 무료/유료로 할 것인가?

기준:
- 핵심 루프 = 무료
- 양적 확장 (더 많은 퀘스트, 더 많은 인증) = 무료
- 사회적 연결 (디스코드, 라이브, 1:1) = 유료
- 창업자 시간이 들어가는 것 = 유료

### 가격을 올릴 것인가?

기준:
- 다음 정원이 지난 정원보다 빨리 차면 → 가격 인상
- 이탈률이 5% 이하로 안정되면 → 가격 인상 가능
- 단, **기존 멤버의 가격은 절대 올리지 않음** (약속)

## 8.3 실패 신호

다음 중 하나라도 발생하면 멈추고 재검토:

- D7 재방문율이 3개월 연속 25% 미만
- Founder's Circle 월 이탈률이 10% 초과
- 시즌 코호트 4주 완주율이 50% 미만
- "이거 왜 돈 내야 해요?"라는 질문이 5명 이상에게서 나옴
- 창업자 본인이 LV UP을 안 쓰기 시작함

마지막 5번이 가장 위험한 신호다.

# 9. 절대 하지 않을 것

LV UP의 비즈니스 모델에서 **하지 않을 것**을 명확히 한다.

## 9.0 사용자를 가두지 않음 (운영원칙 0번)

> **LV UP은 사용자가 LV UP을 떠나도 더 잘 살 수 있게 만드는 것을 목표로 한다.**
> **떠날 수 없게 만드는 모든 메커니즘은 LV UP의 가장 큰 실패다.**

이것이 9장 전체를 관통하는 0번 원칙이다. 아래 9.1~9.9의 모든 항목은 
이 원칙의 구체적 적용이다. BUSINESS_PLAN.md 13.0과 짝을 이룬다.

### 구체적 실천

- **떠나는 사용자를 잡는 장치 금지**: 환불 거절, 죄책감 푸시, 해지 방해 금지 (9.9 참조)
- **자발적 휴면을 처벌하지 않음**: 레벨/스탯이 떨어지지 않음, "그리워요" 푸시 금지
- **대체 도구를 막지 않음**: LV UP 대체 시스템 언급 허용, 오히려 권장
- **의존성을 성공으로 착각하지 않음**: D7 재방문율 높음 ≠ 성공

### 측정 지표

이 원칙이 실제 약속이 되려면 측정 가능해야 한다.
상세는 PRD Part 21.3 "건강한 이탈 지표" 참조.

**Phase 1 가벼운 지표:**
- 환불률 5% 이하
- 환불 요청 처리 5분 이내
- 휴면 복귀율 (참고 지표)

**Phase 2~ 깊은 지표:**
- 이탈 후 30일 행동 유지율
- 자기 시스템 형성률
- 환불 후 6개월 만족도

### 해석 원칙

**이탈률이 낮은 게 자랑이 아니라, 이탈한 사람의 인생이 LV UP 사용 전보다 
좋아진 비율이 자랑이다.**

D7 재방문율이 높은데 이탈 후 행동 유지율이 낮다면 → 의존 경고.
D7 재방문율이 낮은데 이탈 후 행동 유지율이 높다면 → LV UP이 성공적으로 
사용자를 자립시킨 신호. 두 지표를 함께 봐야 한다.

## 9.1 광고 없음

- 앱 안에 광고 배너 없음
- 광고 영상 없음
- 광고 후원 콘텐츠 없음

**이유**: 광고는 사용자를 상품으로 만든다. LV UP은 사용자가 고객이지 상품이 아니다.

## 9.2 데이터 판매 없음

- 사용자 데이터를 제3자에게 판매하지 않음
- 행동 데이터를 광고주에게 제공하지 않음
- 익명 통계도 외부 판매하지 않음

**이유**: 신뢰가 무너지면 LV UP의 모든 것이 무너진다.

## 9.3 다크 패턴 없음

- 구독 해지를 어렵게 만들지 않음
- "한 번만 더 생각해보세요" 식의 압박 없음
- 자동 갱신 전 명확한 알림
- 환불 요청 시 5분 이내 처리

**이유**: 사람을 가두는 것은 LV UP의 5대 원칙 (자율성)에 위배.

## 9.4 가격 차별 사기 없음

- "오늘만 50% 할인" 같은 가짜 카운트다운 없음
- 동일 사용자에게 다른 가격 제시 없음
- A/B 테스트에서 가격은 테스트하지 않음 (윤리적 이유)

## 9.5 첫 멤버에게 한 약속은 영원히

- 첫 30명 Founder's Circle 멤버 → 평생 9,900원
- 가격이 올라도, 회사가 커져도, 약속은 지킨다
- 이게 LV UP의 신뢰 자산

## 9.6 검증 안 된 것을 팔지 않음

- 작동하는지 모르는 기능을 미리 팔지 않음
- "곧 출시 예정" 기반 결제 받지 않음
- 매출은 검증의 결과여야 한다

## 9.7 사용자를 친구의 친구로 가두지 않음

- LV UP을 안 써도 친구 관계가 끊기지 않게
- 이탈한 사용자에게 죄책감 주지 않음
- 떠난 사람도 환영받는 분위기

## 9.8 창업자 본인이 안 쓰는 것을 만들지 않음

- 모든 기능은 창업자가 먼저 사용
- 안 쓰는 기능은 만들지 않음
- 만든 사람이 안 쓰는 것을 고객이 쓸 리 없다

## 9.9 환불 거부 없음

결정 9에서 부록 A 사고 실험 3을 정식 항목으로 승격.
환불 정책이 곧 9.0 운영원칙 0번의 가장 구체적인 실천이다.

### 규칙

- **환불 요청 시 5분 이내, 무조건 처리**
- **사유 묻지 않음**
- **재결제 유도 금지** ("한 번만 더 생각해보세요" 금지)
- **환불 사용자에 대한 죄책감 메시지 금지**

### 왜 무조건 환불인가

- 안 썼다는 건 우리가 가치를 못 줬다는 뜻이다
- 억지로 돈을 가져가면 평생 적이 된다
- 환불해주면 평생 친구가 된다
- LV UP은 단기 매출보다 장기 신뢰가 100배 중요하다

### 시스템 구현

- Founder's Circle 해지 버튼은 결제 버튼과 같은 크기, 같은 접근성
- 해지 플로우에 "재고려" 단계 없음 — 한 번 클릭으로 완료
- 환불 완료 이메일은 담담한 톤. "언제든 돌아와도 환영이에요" 한 줄로 충분
- 환불 후 6개월간 마케팅 이메일 발송 금지

### 측정

- 환불률: 5% 이하 목표 (참고 지표일 뿐, 낮추는 게 목적 아님)
- 환불 요청 → 처리까지 평균 시간: 5분 이내
- 환불 후 재가입률: 별도 추적 (참고 지표)
- **환불 후 6개월 만족도 조사** (Phase 2~): "환불하신 결정에 후회하지 
  않으시나요?" — 이 답이 '아니오'이면 LV UP의 승리

## 10.3 회고의 원칙

**1. 정직하게 쓴다**
- 안 됐으면 안 됐다고 쓴다
- 좋은 척하지 않는다
- 나중에 보면 가장 정직한 회고가 가장 가치 있다

**2. 사용자의 말을 인용한다**
- "사용자의 한 마디"가 가장 중요한 데이터
- 숫자보다 정확할 때가 많다

**3. 가설을 죽이는 것을 두려워하지 않는다**
- 가설이 틀렸다는 것을 빨리 발견할수록 이득
- 이 문서의 목적은 정답이 아니라 학습

**4. 한 페이지를 넘지 않는다**
- 길게 쓰면 안 본다
- 핵심만, 간결하게

# 부록 A. 가격 결정의 사고 실험

LV UP의 가격을 어떻게 정해야 할지 막막할 때 참고하는 사고 실험.

## 사고 실험 1: 한 잔의 커피

> 사용자가 LV UP에 한 달에 9,900원을 낸다는 것은,
> 한 달에 스타벅스 라떼 1잔을 안 마시고 LV UP에 투자한다는 뜻이다.
>
> LV UP이 그 한 잔의 라떼보다 가치 있는가?

라떼보다 가치 있어야 한다. 그래야 가격이 정당하다.

## 사고 실험 2: 친한 친구

> 사용자가 99,000원짜리 시즌 코호트를 결제했다.
> 친한 친구가 "그거 왜 샀어?"라고 묻는다.
> 사용자는 뭐라고 대답할 것인가?

대답이 명확해야 한다. 모호하면 가격이 비싸다.

좋은 대답: "한 달 동안 같이 인생 바꿔보려고."
나쁜 대답: "그냥... 좋아 보여서."

## 사고 실험 3: 환불 요청

> 사용자가 결제 후 30일이 지나서 환불을 요청한다.
> "별로 안 썼어요."
>
> 환불해줄 것인가?

답: 무조건 환불.

이유:
- 안 썼다는 건 우리가 가치를 못 줬다는 뜻
- 억지로 돈을 가져가면 평생 적이 됨
- 환불해주면 평생 친구가 됨
- LV UP은 단기 매출보다 장기 신뢰가 100배 중요

## 사고 실험 4: 가격을 두 배로 올리면

> 가격을 9,900원 → 19,900원으로 올리면 멤버 수가 절반이 될 것인가?

만약 그렇다면 → 가격 인상 의미 없음 (총매출 동일)
만약 절반보다 덜 떨어진다면 → 가격을 올릴 가치 있음
만약 절반보다 더 떨어진다면 → 현재 가격이 적절

이걸 실제 실험으로 확인할 수 있는 건 Phase 2 후반.
