# Part 1. 제품의 본질

## 1.1 한 줄 정의

**LV UP은 현실에서 행동한 것을 인증하고, 그 실행이 상태창의 레벨과 능력치로 남는 성장 RPG형 웹앱이다.**

## 1.2 더 깊은 정의

LV UP은 당신을 자기 인생의 CEO로 만든다.
거대한 목표를 작은 퀘스트로 쪼개서 하루에 하나씩 실행하고,
그 즉시 성장의 도파민을 느끼며,
매출과 선한 영향력을 통해 자아실현이라는 가장 높은 욕구까지 도달하게 만든다.

## 1.3 핵심 루프

**행동 → 인증 → 보상 → 성장 기록 → 다음 행동**

이 단순한 5단계 루프가 LV UP의 모든 것이다.
복잡한 기능은 다 나중에 와도 된다. 이 루프 하나만 제대로 작동하면 된다.

## 1.4 핵심 가치

- 사용자는 할 일을 관리하지 않는다. **퀘스트를 수행한다**.
- 성장 결과는 텍스트 목록이 아니라 **상태창**으로 보인다.
- 행동은 말이 아니라 **이미지/기록 인증**으로 남는다.
- 보상은 숫자만이 아니라 **서사적 메시지**와 함께 제공된다.
- 개인 성장에서 시작하지만, 이후에는 **파티/길드/프로젝트 협업**으로 확장된다.

# Part 2. 사용자가 LV UP을 선택해야 할 이유

이 섹션이 모든 결정의 우선 기준이다.
"사용자가 정말 하고 싶게 만드는 것"이 목적이라면, 그 답이 여기 있어야 한다.

## 2.1 LV UP은 누구를 위한 것인가

**핵심 사용자 (Phase 1)**

- 20~40대
- 자기계발, 창작, 개발, 창업, 프로젝트 실행 욕구가 강함
- 계획은 자주 세우지만 실행 지속이 어려움
- 일반 습관앱/투두앱의 차가운 UX에 피로감을 느낌
- "내 상태창", "내 성장 유형", "내 클래스" 같은 자기 해석형 경험을 좋아함

**확장 사용자 (Phase 2 이후)**

- 협업 프로젝트를 만들고 싶은 사람
- 공익성/사회적 기여 활동을 게임처럼 하고 싶은 사람
- 커뮤니티 안에서 피드백, 응원, 결과물 공유를 원하는 사람

## 2.2 사용자가 매일 돌아오게 만드는 5가지 장치

1. **자기 해석의 즐거움**: "오늘 내 상태창이 어떻게 변했지?" — 자기 자신을 게임 캐릭터처럼 보는 새로운 시각
2. **즉각적 보상의 도파민**: 인증 후 0.5초 안에 시각적/서사적 피드백
3. **연속 수행의 매몰비용**: 7일 연속 → 30일 연속, 한 번 끊기기 싫은 시각적 단서
4. **다음의 호기심**: "내일 어떤 퀘스트가 열릴까?", "다음 레벨에서 무엇이 해금될까?"
5. **작은 인정의 따뜻함**: 누군가 내 인증에 박수를 보내거나, 내 정리가 도움됐다는 신호 (Phase 2)

## 2.3 사용자가 LV UP에서 절대 느끼면 안 되는 5가지

1. **압박감**: "오늘도 못 했네"라는 죄책감을 주면 끝이다. 처벌 대신 회복을 설계.
2. **귀찮음**: 인증이 5분 이상 걸리면 안 된다. 사진 한 장이면 충분해야 한다.
3. **차가움**: "+10 XP" 같은 건조한 숫자만 보여주면 안 된다.
4. **비교의 굴욕**: 랭킹과 서열은 Phase 1에 절대 없다. 자기 자신과의 비교만 있다.
5. **공허함**: 꾸준히 했는데 "그래서 뭐?"라는 느낌이 들면 끝이다. 누적이 의미 있게 느껴져야 한다.

# Part 4. LV UP 5대 차별화 원칙

모든 기능, 화면, 메시지는 이 5원칙에 부합해야 한다.

### 원칙 1: 도구가 아니라 세계

사용자는 도구를 설정하지 않는다. 이미 살아있는 세계에 입장한다.
클래스 하나만 고르면 첫 퀘스트가 자동으로 나오고, 진행하면 다음이 풀린다.

### 원칙 2: 숫자가 아니라 이야기

"+5 EXE" 대신 "오늘 너는 한계를 한 발짝 넘었다. 실행력이 강화되었다."
모든 보상은 작은 서사적 사건으로 표현된다.

### 원칙 3: 스펙이 아니라 행동

학력, 자격증, 자산은 LV UP의 본질이 아니다.
오늘 무엇을 했고, 내일 무엇을 할 것인가가 모든 것이다.

### 원칙 4: 비교가 아니라 자기 성장

랭킹과 서열은 없다. 어제의 나와 오늘의 나만 비교한다.
타인은 비교 대상이 아니라 함께 가는 동료다.

### 원칙 5: 성장에는 멈춤도 포함된다

"더 빨리"가 아니라 "건강하게".
강제 휴식 퀘스트, "오늘은 충분히 했다" 메시지, 만족 인증.
도파민 루프가 번아웃이 아니라 자아실현으로 향하게 설계.

# Part 5. 메인 스탯 6개

LV UP의 상태창은 메인 스탯 6개로 구성한다.
누구나 공통으로 갖는 인간의 기본 성장 축이며, 모든 퀘스트는 이 6개 중 하나 이상에 연결된다.

## 5.1 6대 메인 스탯

### 1. 체력 | Vitality

- **정의**: 몸을 움직이고 버티고 회복하는 힘
- **올리는 행동**: 운동, 산책, 수면 관리, 식습관 관리, 건강 루틴 유지
- **상징 컬러**: 붉은 오렌지 (#f97316)

### 2. 집중력 | Focus

- **정의**: 주의를 모으고 몰입을 유지하는 힘
- **올리는 행동**: 집중 작업, 딥워크, 방해 없이 공부하기, 몰입 시간 측정
- **상징 컬러**: 깊은 청록 (#0891b2)

### 3. 실행력 | Execution

- **정의**: 생각을 행동으로 옮기고 끝까지 밀어붙이는 힘
- **올리는 행동**: 할 일 완료, 미루던 일 시작, 데드라인 준수, 프로젝트 전진
- **상징 컬러**: 인디고 (#6366f1)

### 4. 지식력 | Knowledge

- **정의**: 배우고 이해하고 조사하고 해석하는 힘
- **올리는 행동**: 독서, 강의 수강, 조사, 분석, 기록 정리, 학습 공유
- **상징 컬러**: 보랏빛 (#a855f7)

### 5. 관계력 | Relationship

- **정의**: 사람과 건강하게 연결되고 함께 성장하게 만드는 힘
- **올리는 행동**: 협업, 피드백 주고받기, 공감하기, 상담/조언, 응원, 조율
- **상징 컬러**: 따뜻한 연두 (#84cc16)

### 6. 전파력 | Influence

- **정의**: 내 생각, 지식, 결과물을 다른 사람에게 전달하고 퍼뜨리는 힘
- **올리는 행동**: 글쓰기, 발표, 설명, 콘텐츠 공유, 노하우 전달, 커뮤니티 기여
- **상징 컬러**: 황금빛 (#eab308)

## 5.2 메인 스탯 설계 원칙

- 사용자가 이름만 보고 즉시 이해할 수 있어야 한다.
- 어떤 퀘스트가 어느 스탯을 올리는지 직관적이어야 한다.
- 스펙이 아니라 **행동과 기여**가 올라가야 한다.
- 6개 스탯의 균형이 좋다고 강제하지 않는다. **편향이 곧 정체성**이다.

## 5.3 상태창 표시 방식

각 스탯은 아래 요소를 함께 노출한다.

- 숫자 값
- 최근 7일 변화량 (▲5 같은 형식)
- 현재 상태 해석 문구

**예시:**

```
체력 18 · 회복 단계 · ▲2
집중력 24 · 안정적 · ▲3
실행력 31 · 가속 중 · ▲7
지식력 22 · 꾸준함 · ▲4
관계력 14 · 잠잠함 · ▲0
전파력 12 · 시작 전 · ▲1
```

## 5.4 상태창 해석의 재미

해석 문구는 단순 숫자보다 100배 강력하다. 사용자는 자기 상태창을 보며 자기 자신을 새롭게 읽게 된다.

**해석 예시:**

- 체력 높고 집중력 낮음 → "회복은 되지만 몰입이 약함"
- 실행력 높고 관계력 낮음 → "혼자 잘하나 협업이 약함"
- 지식력과 전파력 모두 높음 → "배우고 전달하는 타입"
- 6개 모두 균형 → "전천후 성장형"
- 1~2개에 강하게 편향 → "스페셜리스트 성향"

이 해석 자체가 사용자가 매일 돌아오는 이유 중 하나가 된다.

# Part 6. 클래스 5개

클래스는 직업이 아니라 **문제를 푸는 방식**, 플레이 스타일이다.

## 6.1 5대 클래스

### 1. 제작자 | Builder

- **설명**: 손으로 만들고 구현하는 사람
- **강한 축**: 실행력, 집중력
- **어울리는 역할**: 개발, 설계, 제작, 자동화, 구축
- **상징 이미지**: 도구를 든 손, 청사진, 망치
- **컬러**: 인디고

### 2. 창작자 | Creator

- **설명**: 표현하고 창조하는 사람
- **강한 축**: 전파력, 지식력
- **어울리는 역할**: 디자인, 브랜딩, 카피, 영상, 콘텐츠
- **상징 이미지**: 펜, 캔버스, 빛나는 별
- **컬러**: 보랏빛

### 3. 주도자 | Leader

- **설명**: 방향을 잡고 끌고 가는 사람
- **강한 축**: 실행력, 관계력
- **어울리는 역할**: PM, 기획, 운영 리드, 팀 리딩
- **상징 이미지**: 깃발, 나침반, 지휘봉
- **컬러**: 황금빛

### 4. 탐구자 | Explorer

- **설명**: 배우고 조사하고 밝혀내는 사람
- **강한 축**: 지식력, 집중력
- **어울리는 역할**: 리서치, 조사, 분석, 전략
- **상징 이미지**: 망원경, 책, 지도
- **컬러**: 청록

### 5. 조율자 | Supporter

- **설명**: 돕고 연결하고 팀을 살리는 사람
- **강한 축**: 관계력, 체력
- **어울리는 역할**: 운영, 커뮤니케이션, 피드백, 조정, 지원
- **상징 이미지**: 손잡은 두 사람, 다리, 따뜻한 빛
- **컬러**: 연두

## 6.2 클래스 설계 원칙

- 클래스는 사용자를 좁은 직군으로 가두지 않는다.
- 클래스는 **성향**이며, 역할 태그는 **실무 역할**이다.
- 가입 시 직접 선택. 진단 테스트는 후순위.
- 나중에 행동 패턴 기반으로 "당신은 사실 탐구형에 가깝습니다" 같은 리포트는 Phase 2 기능.

## 6.3 가입 시 클래스 선택 UX

1. 5개 카드를 한 화면에 (모바일에선 좌우 스와이프)
2. 각 카드에 한 줄 설명 + 대표 아이콘 + 컬러
3. 탭하면 즉시 선택, 확인 버튼 없음
4. 선택 시 짧은 글로우 애니메이션
5. "당신의 클래스는 [제작자]입니다. 손으로 만드는 자의 길이 시작됩니다." 같은 서사적 메시지

# Part 7. 역할 태그

역할 태그는 파티/프로젝트 안에서의 **실무 역할**이다.
클래스(성향)와는 별개로 다중 선택 가능.

## 7.1 초기 역할 태그 (Phase 1)

기본 13개:

- 기획
- 디자인
- 개발
- 마케팅
- 운영
- 리서치
- 분석
- 글쓰기
- 편집
- 설계
- 세일즈
- 교육
- 커뮤니케이션

확장 가능: 요리, 운동, 건축, 음악, 영상, 사진 등

## 7.2 설계 원칙

- 클래스는 1개만 선택
- 역할 태그는 1~3개 선택 가능
- 나중에 파티 모집/프로젝트 매칭 시 활용
- 사용자가 새로운 태그를 제안할 수 있게 함 (Phase 2)

**예시 조합:**

- 클래스: 제작자 / 역할: 설계, 개발, 자동화 → "혼자서 시스템을 짓는 사람"
- 클래스: 창작자 / 역할: 디자인, 글쓰기 → "비주얼과 텍스트를 모두 다루는 사람"
- 클래스: 조율자 / 역할: 운영, 커뮤니케이션 → "팀의 윤활유"

# Part 9. 3단 XP 보상 구조

LV UP의 보상은 단일 XP가 아니라 3단으로 설계한다.
이게 LV UP을 단순 습관앱과 결정적으로 가르는 메커니즘이다.

## 9.1 수행 XP — 행동 자체에 대한 보상

행동을 했는가? → 기본 XP 지급

**예시:**
- 운동 완료 → +15 XP
- 글 작성 완료 → +20 XP
- 요리 결과물 업로드 → +15 XP
- 프로젝트 한 단계 전진 → +30 XP

## 9.2 증빙 XP — 기록의 깊이에 대한 보상

기록을 더 잘 남겼는가? → 추가 XP 지급

**예시:**
- 과정 사진 추가 → +5 XP
- 한 줄 소감/정리 → +5 XP
- Before/After 기록 → +10 XP
- 상세 설명 첨부 → +10 XP

## 9.3 기여 XP — 타인에게 도움이 된 것에 대한 보상

다른 사람에게 도움이 되었는가? → 보너스 XP 지급 (Phase 2)

**예시:**
- 레시피 공유가 유용함 평가 → +20 XP
- 글이 "도움됨" Verdict 획득 → +15 XP
- 정리한 정보가 다른 사람에게 참고됨 → +10 XP

## 9.4 왜 이게 결정적인가

이 3단 구조는 사용자에게 **세 가지 다른 차원의 성장**을 동시에 제공한다.

1. **수행 XP**: "나는 했다" — 행동의 만족
2. **증빙 XP**: "나는 기록한다" — 자기 인식의 깊이
3. **기여 XP**: "나는 도움이 된다" — 사회적 의미

매슬로우 욕구 단계로 보면:
- 수행 XP = 안전/소속의 욕구 충족 (일상의 안정)
- 증빙 XP = 존중의 욕구 충족 (자기 효능감)
- 기여 XP = 자아실현의 욕구 충족 (의미 있는 영향)

**LV UP이 단순 도파민 루프가 아니라 진짜 자아실현의 사다리가 되는 메커니즘이 여기 있다.**

# Part 10. 인증 시스템

## 10.1 기본 원칙

- 인증은 **간편해야** 한다 (5분 이내)
- 동시에 "진짜 했다"는 **현실감**이 있어야 한다
- 업로드 스트레스가 너무 크면 안 된다

## 10.2 권장 구조

- 대표 이미지 1장 (권장, 필수 아님)
- 텍스트 메모 1줄 이상 (선택)
- 추가 이미지 첨부 (선택, 최대 10장)
- 링크 첨부 (선택, GitHub/Notion/블로그 등)

## 10.3 이미지 정책

- 대표 이미지 1장 우선 노출 (피드, 로그, 상태창 미리보기)
- 추가 이미지는 상세 페이지에서만 노출
- 1퀘스트당 최대 10장
- R2에 저장, presigned URL 또는 서버 경유

## 10.4 인증 없는 완료도 허용

이게 중요한 디테일이야. 모든 퀘스트에 사진을 강제하면 사용자가 도망간다.

- 완료 체크만 해도 수행 XP는 지급
- 인증을 추가하면 증빙 XP 보너스
- "해야 한다"가 아니라 "하면 더 좋다"

# Part 11. 서사적 보상 메시지 시스템

## 11.1 핵심 원칙

퀘스트 완료 시 단순 숫자가 아닌, 세계관이 살아있는 메시지를 표시한다.
이게 LiFE RPG와 LV UP을 가르는 가장 중요한 차이다.

## 11.2 스탯별 메시지 풀

각 스탯마다 5~10개의 메시지를 만들어두고 랜덤으로 표시한다.

### 체력 | Vitality
- "오늘 너는 몸을 깨웠다. 내일의 너에게 선물이 되었다."
- "체력은 모든 능력의 그릇이다. 그릇이 한 뼘 자랐다."
- "한 발 더 뛴 자만이 다음 산을 본다."

### 집중력 | Focus
- "산만한 세상에서 한 점에 머무는 것은 가장 어려운 기술이다."
- "오늘 너의 시간은 흩어지지 않았다. 집중의 룬이 새겨졌다."
- "20분의 몰입이 한 시간의 산만함을 이긴다."

### 실행력 | Execution
- "오늘 너는 한계를 한 발짝 넘었다. 실행력이 강화되었다."
- "생각만 하는 자와 실행하는 자 사이에 우주가 있다. 너는 우주를 건넜다."
- "완벽한 시작은 없다. 시작한 자만이 도착한다."

### 지식력 | Knowledge
- "새로운 지식이 시야를 넓혔다. 아직 읽지 않은 책장이 빛나고 있다."
- "오늘 알게 된 것이 내일의 너를 다르게 만들 것이다."
- "배우는 자는 어제의 자신을 매일 추월한다."

### 관계력 | Relationship
- "새로운 연결이 생겼다. 세계는 혼자 넓히는 것이 아니다."
- "오늘의 한마디가 누군가의 하루를 바꿨을 것이다."
- "혼자 가면 빠르고, 함께 가면 멀리 간다. 너는 멀리 가는 길을 택했다."

### 전파력 | Influence
- "네 목소리가 한 사람에게 닿았다. 영향력은 거기서 시작된다."
- "오늘 네가 쓴 것이 내일 누군가의 등불이 될지 모른다."
- "전한다는 것은 가장 큰 학습이다. 너는 두 번 배웠다."

## 11.3 레벨업 메시지

레벨업 시에는 전체 화면 연출과 함께 특별한 메시지를 표시한다.

```
── LEVEL UP ──

당신은 더 이상 어제의 당신이 아니다.

Lv.5 → Lv.6
"길을 찾은 자"

새로운 퀘스트가 해금되었습니다.
```

| 레벨 | 칭호 | 의미 | 해금 |
|------|------|------|------|
| 1 | 입장한 자 | 첫 발을 디딘 사람 | 기본 퀘스트 |
| 2 | 자각한 자 | 자신을 보기 시작한 사람 | 커스텀 퀘스트 생성 |

# Part 12. 뱃지와 자격증

## 12.1 뱃지의 역할

- 인증 표시
- 기여 인정
- 숙련도 상징
- 상위 칭호

## 12.2 뱃지 유형

### 인증 뱃지
- 설계 인증, 개발 인증, 조리 인증, 실무 인증

### 기여 뱃지
- 도움을 준 자, 꾸준한 기록자, 좋은 설명가, 길을 밝힌 자

### 숙련 뱃지
- 글쓰기 숙련자, 설계 숙련자, 요리 숙련자

### 상위 칭호 뱃지
- 초급 → 숙련 → 전문가 → 마스터 → 태양신

## 12.3 자격증 처리 원칙

- 자격증은 메인 성장축이 아니다
- 자격증은 해당 분야 스킬의 **신뢰 보조 장치**로만 사용
- 자격증 때문에 메인 스탯이 폭증하지 않게 함
- 자격증 인증 시 해당 분야 인증 뱃지 부여

**예시:**
- 한식조리기능사 → 요리 인증 뱃지
- 정보처리기사 → 개발 인증 뱃지
- 건축사 → 설계 인증 뱃지

# Part 13. Verdict 시스템 (Phase 2 우선, 구조는 미리 정의)

## 13.1 핵심 정의

Verdict는 사람을 심사하는 시스템이 **아니다**.
**결과물의 유익함과 기여도를 인정하는 추가 보상 시스템**이다.

## 13.2 Verdict가 붙는 콘텐츠

- 글, 레시피, 설계 팁
- 개발 노하우, 운동 루틴
- 독서 요약, 임장 정리

## 13.3 Verdict 유형

- 도움됨
- 잘 정리됨
- 실전적임
- 신뢰됨
- 영감됨

## 13.4 Verdict의 역할

- 기본 XP를 막는 검수 시스템이 **아님**
- 기본 XP와 기본 스탯 보상은 Verdict와 **무관하게 즉시 지급**
- 좋은 결과물에 **기여 XP를 보너스로** 지급
- 추후 커뮤니티형 인정 시스템과 결합

## 13.5 분야별 신뢰도 기반 Verdict 가중치 (Phase 2 확장)

Verdict는 향후 단순 반응 시스템이 아니라, **해당 분야에서 신뢰도가 높은 사용자의 평가가 더 큰 보너스를 갖는 구조**로 확장한다.
이게 LV UP을 단순 좋아요/추천 시스템과 결정적으로 가르는 메커니즘이다.

### 13.5.1 핵심 원칙

- 기본 XP/스탯 보상은 Verdict와 **무관하게 즉시 지급** (행동한 사람을 처벌하지 않는다)
- Verdict는 깎는 시스템이 **아니라 더하는 시스템**
- Verdict 가중치는 **총 레벨이 아니라 해당 분야의 신뢰도**로 계산
- 한 명의 고신뢰 사용자가 전체 결과를 좌우하지 않도록 **상한선** 적용
- "고인물 권력화"를 막는 것이 설계의 최우선 과제

### 13.5.2 왜 "총 레벨"이 아니라 "분야별 신뢰도"인가

이게 가장 중요한 차별점이다.

대부분의 평가 시스템은 "활동 많은 사람"이 모든 분야의 평가권을 갖는다.
이게 결국 고인물이 신규를 짓누르는 구조를 만든다.

LV UP은 다르다.
**운동을 잘하는 사람의 운동 평가만 무겁다.**
글을 잘 쓰는 사람의 글 평가만 무겁다.
요리 경험이 많은 사람의 요리 평가만 무겁다.

자기 분야가 아닌 곳에서는 모두가 "초보 평가자"다.
이게 권력의 분산이고, LV UP의 평등을 지키는 핵심 장치다.

### 13.5.3 분야별 신뢰도 구성 요소

각 분야의 신뢰도는 아래 5가지 요소의 조합으로 계산한다.

1. **분야 스킬 수준** — 해당 분야에서 직접 쌓은 숙련도
2. **관련 메인 스탯** — 그 분야와 연결된 메인 스탯의 수치
3. **인증 뱃지** — 자격증/실무 인증 등으로 받은 신뢰 보조 표식
4. **누적 퀘스트 기록** — 해당 분야 퀘스트를 얼마나 꾸준히 수행했는가
5. **과거 Verdict의 정확성** — 과거에 한 평가가 시간이 지나도 유효했는가

### 13.5.4 분야별 적용 예시

#### 운동 분야

**신뢰도 구성:**
- 운동 스킬 수준
- 체력, 집중력, 실행력 메인 스탯
- 운동 관련 인증 뱃지 (트레이너 자격, 실무 경력 등)
- 운동 퀘스트 누적 수행 기록
- 과거 운동 Verdict의 유용성/정확성

**Verdict 항목:**
- 꾸준함
- 실전성
- 안전성
- 설명력
- 초보자 친화성
- 동기부여

**적용 예시:**
- 일반 사용자의 운동 Verdict → 낮은 가중치 보너스 XP
- 운동 분야 신뢰 평가자의 Verdict → 높은 가중치 보너스 XP
- 운동 전문 평가자(자격+경력+꾸준함 모두)의 Verdict → 가장 높은 가중치

#### 글쓰기 분야

**신뢰도 구성:**
- 글쓰기 스킬, 지식력, 전파력
- 글쓰기 관련 누적 결과물
- 과거 글쓰기 Verdict의 정확성

**Verdict 항목:**
- 명료함
- 통찰력
- 가독성
- 진정성
- 새로움
- 도움됨

#### 요리 분야

**신뢰도 구성:**
- 요리 스킬, 체력, 실행력
- 요리 관련 인증 (조리사 자격 등)
- 요리 퀘스트 누적 기록

**Verdict 항목:**
- 재현성 (따라하기 쉬움)
- 맛 (먹어본 사람의 평가)
- 가성비
- 건강함
- 창의성
- 설명의 친절함

### 13.5.5 보상 구조와의 연결

| 단계 | 받는 보상 | 누가 결정 |
|------|----------|-----------|
| 행동 자체 | 수행 XP + 스탯 | 자동 지급 (Verdict 무관) |
| 기록 추가 | 증빙 XP | 자동 지급 (Verdict 무관) |
| 일반 평가 | 일반 가중치 기여 XP | 모든 사용자 |
| 신뢰자 평가 | 높은 가중치 기여 XP | 분야 신뢰도 보유자 |
| 전문가 평가 | 최고 가중치 기여 XP | 분야 최고 신뢰도 보유자 |

핵심: **하위 단계가 상위 단계를 차단하지 않는다.** 모든 사용자는 행동만 해도 즉시 보상을 받고, Verdict는 그 위에 쌓이는 보너스다.

### 13.5.6 설계 시 절대 피해야 할 패턴

이 5가지는 **반드시 막아야 하는 안티패턴**이다.

1. **고인물 권력화**
   - 한 사람이 너무 많은 영향력을 갖지 않도록 상한선 필수
   - 같은 평가자의 같은 사용자에 대한 반복 평가 가중치 감소

2. **신규 사용자의 참여감 저하**
   - 신규 사용자도 일반 가중치로 평가에 참여 가능
   - "신뢰도 낮으면 평가도 못 함"은 절대 금지

3. **분야 외 영향력 누수**
   - 운동 신뢰도가 높다고 글쓰기 평가 가중치가 올라가지 않음
   - 신뢰도는 분야별로 완전히 분리

4. **친목/품앗이 악용**
   - 같은 사람들끼리 서로 평가해주는 패턴 탐지
   - 다양한 사용자에게 평가받을수록 가중치 상승

5. **보상 지연**
   - Verdict가 늦게 붙어도 기본 보상은 즉시
   - "심사 대기"라는 개념 자체가 없어야 함

### 13.5.7 MVP에서의 위치

이 시스템은 **Phase 1 MVP에는 포함하지 않는다.**

Phase 1에서는:
- 수행 XP + 증빙 XP까지만 작동
- 기여 XP는 시스템에는 있되 (DB 필드 존재) 활성화되지 않음
- Verdict 자체가 P2 기능

Phase 2에서 도입할 때도:
- 처음엔 단순 일반 가중치로만 시작
- 분야별 신뢰도 시스템은 충분한 사용자/데이터가 쌓인 후에 활성화

이 구조는 지금 **데이터 모델과 철학을 확정**해두기 위한 것이다.
실제 구현은 LV UP이 충분히 성숙한 후의 일이다.

# Part 20. XP & 레벨 산정 규칙

## 20.1 레벨업 곡선

```
Lv.1 → 2:  100 XP
Lv.2 → 3:  150 XP
Lv.3 → 4:  225 XP
Lv.4 → 5:  340 XP
Lv.5 → 6:  500 XP
Lv.6 → 7:  750 XP
...

공식: required_xp = floor(100 * 1.5^(level-1))
```

초반에 빠른 성취감, 후반에 진짜 노력 필요한 곡선.

## 20.2 연속 수행 보너스

| 연속 일수 | 보너스 |
|----------|--------|
| 2일 | +10% XP |
| 7일 | +25% XP + "흔들리지 않는 자" 배지 |
| 30일 | +50% XP + 특별 칭호 |
| 100일 | +75% XP + 전설급 칭호 |

**중요**: 연속이 끊겨도 레벨/스탯은 떨어지지 않음. **처벌이 아닌 보상 설계.**

## 20.3 스탯 반영

| 난이도 | 스탯 증가 |
|--------|----------|
| Easy | +1 |
| Normal | +2 |
| Hard | +3 |
| Epic | +5 |

스탯은 단순 합산 (Phase 1). 가중치/감소는 Phase 2에서 검토.
